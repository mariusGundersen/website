Pan Zoom

Input

The input to our pan-zoom component should be for both computers with a mouse and for devices with a touch screen. We could implement it using mouse events for the computer and touch events for the mobile device, but there is a better and more modern solution in pointer events (not to be confused with css pointer events). Lets start by attaching events listeners to the container element.

container.addEventListener('pointerdown', e => {
}, false);

container.addEventListener('pointermove', e => {
}, false);

container.addEventListener('pointerup', e => {
}, false);

Before we go further we need to make sure that we can only pan-zoom the container element, and that the browser doesn't attempt to be helpful. For example, on a touch device a pan could be interpreted as scroll. We can prevent these default actions using a single css line:

container.style.touchAction = 'none';

Next we need to handle correctly the pointer moving only when it is down, but also potentially moving outside the container element. We could have added the pointermove event to the document, and we could have tracked if the pointer is down using some state variable, but there is a simpler way.

container.addEventListener('pointerdown', e => {
  container.setPointerCapture(e.pointerId);
}, false);

container.addEventListener('pointermove', e => {
  if(continer.hasPointerCapture(e.pointerId)){
    // ...
  }
}, false);

container.addEventListener('pointerup', e => {
  // pointer capture is implicitly released on pointerup,
  // so we don't need to do anything here
}, false);

Notice that we now don't need to call e.preventDefault() or e.stopPropagation(), like we would have done with a mouse or touch event.

Now we need to find out where in the container the pointer is pointing. This is something that we will need in multiple event listeners, so let's make a simple function for it

container.addEventListener('pointerdown', e => {
  container.setPointerCapture(e.pointerId);
  const viewPos = getViewPos(e);
}, false);

container.addEventListener('pointermove', e => {
  if(continer.hasPointerCapture(e.pointerId)){
    const viewPos = getViewPos(e);
  }
}, false);

container.addEventListener('pointerup', e => {
}, false);

function getViewPos(event) {
  const rect = event.currentTarget.getBoundingClientRect();

  return {
    x: event.clientX - rect.x,
    y: event.clientY - rect.y
  };
}

We need to keep track of which pointers are active and where they point, so that we can process them. We'll use a map where the key is the pointerId and the value is an object into which we can put any data we want. For now we will store the viewPos.

const pointers = new Map();

container.addEventListener('pointerdown', e => {
  container.setPointerCapture(e.pointerId);
  const viewPos = getViewPos(e);
  pointers.set(e.PointerId, {
    viewPos
  });
}, false);

container.addEventListener('pointermove', e => {
  if(continer.hasPointerCapture(e.pointerId)){
    const viewPos = getViewPos(e);
    const pointer = pointers.get(e.pointerId);
    pointer.viewPos = viewPos;
  }
}, false);

container.addEventListener('pointerup', e => {
  pointers.delete(e.pointerId);
}, false);

function getViewPos(event) {
  const rect = event.currentTarget.getBoundingClientRect();

  return {
    x: event.clientX - rect.x,
    y: event.clientY - rect.y
  };
}

One more thing we need is a way to process the pointers that are currently down. It is recommended to do as little as possible in the event handlers, so we will schedule the processing of the pointers some time in the future, using requestAnimationFrame. This has the added benefit that multiple pointermove events can fire without reprocessing.

const pointers = new Map();
let hasRequestedAnimationFrame = false;

container.addEventListener('pointerdown', e => {
  container.setPointerCapture(e.pointerId);
  const viewPos = getViewPos(e);
  pointers.set(e.PointerId, {
    viewPos
  });
}, false);

container.addEventListener('pointermove', e => {
  if(continer.hasPointerCapture(e.pointerId)){
    const viewPos = getViewPos(e);
    const pointer = pointers.get(e.pointerId);
    pointer.viewPos = viewPos;
    if(!hasRequestedAnimationFrame){
      hasRequestedAnimationFrame = requestAnimationFrame(() => {
        hasRequestedAnimationFrame = false;
        processPointers();
      });
    }
  }
}, false);

container.addEventListener('pointerup', e => {
  pointers.delete(e.pointerId);
}, false);

function processPointers(){
  // ToDo: process the events
}

function getViewPos(event) {
  const rect = event.currentTarget.getBoundingClientRect();

  return {
    x: event.clientX - rect.x,
    y: event.clientY - rect.y
  };
}

This is a good start, we are now ready to handle these pointer events, process the input and produce some output.

We can figure out where in the view a point will be by transforming it from the model to the view using a transform. Or, in other words, we can say that the view position is the model position transformed.

v=Tm

In this equation v represents the position in the view, m represents the position in the model and T is the transformation matrix. We can expand this out to show all the values involved.

⎣⎢⎡​xv​yv​1​⎦⎥⎤​=⎣⎢⎡​s00​0s0​xT​yT​1​⎦⎥⎤​⎣⎢⎡​xm​ym​1​⎦⎥⎤​

A few things might be weird here, for example the third row in each of these matrices. Matrix math can be a bit weird, so let's not worry too much about it. The last row is there because it makes the multiplication between the transformation matrix and the position vector possible. Matrix multiplication is also a bit strange, let's see how it works.

xv​yv​1​=sxm​+0ym​+1xT​=0xm​+sym​+1yT​=0xm​+0ym​+1​

The single matrix equation we had above has been multiplied out and results in three equations. There are some terms that can go away, so let's simplify this even more

xv​yv​​=sxm​+xT​=sym​+yT​​

By removing the last row and the terms that are 0 we end up with these two nice equations. They can be represented in code as

function toView(m, t){
  return {
    x: m.x*s + t.x,
    y: m.y*s + t.y
  }
}

But what if we want to go the other way, from the view to the model? For example, when we click somewhere in the view, what point in the model are we clicking on? We can rearrange the equation to get the model coordinates.

xv​−xT​=sxm​yv​−yT​=sym​​

First we move over the translation term

sxv​−xT​​syv​−yT​​​=xm​=ym​​

Then we divide by the scale

xm​ym​​=sxv​−xT​​=syv​−yT​​​

And after flipping the equations we end up with this, which can be represented in code as

function toModel(v, t){
  return {
    x: (v.x - t.x)/s,
    y: (v.y - t.y)/s
  }
}

Finally we can rearrange the equations so that, given a point in the view and a point in the model we can find out how to translate our model so the two points match up. That is, we can solve for the translation

xv​−sxm​=xT​yv​−sym​=yT​​

We move the terms so that our translation x and y values are alone

xT​=xv​−sxm​yT​=yv​−sym​​

And if we flip it around we get a simple solution. One thing to note here is that since we have only two equations we can only solve for the two unknown values x and y, not the scale. In other words, we can only translate, we can't scale. In code it would look like this:

function solve(v, m, s){
  return {
    x: v.x - m.x*s,
    y: v.y - m.y*s,
    s
  }
}
